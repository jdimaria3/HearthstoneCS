Card Searcher

Want to contribute to the project? Learn how here!

GitHub and Installation

To get started, we recommend checking out our GitHub page for this project, which can be found at the link here. Once you've familiarized yourself with the code, feel free to fork it and start editing.

Before you get started, though, you should be aware that this project has a few dependencies that you'll need to install in order to ensure you're able to successfully run and test the code.

  1. Python and Pip3 (for installations)

  2. Flask

  3. Flask_Restful

  4. Flask_CORS

Once you have installed Python 3 and Pip3, you can simply install the Flask dependencies by running “pip3 install [dependency]".

How To Run The Project

After installing all of the needed dependencies, change into the “server” directory and run the server by typing “python3 ./server.py”. You may see something like this: 

        Serving Flask app "server" (lazy loading)
        Environment: production
        Debug mode: on
        Running on http://127.0.0.1:8080/ (Press CTRL+C to quit)
        Restarting with stat
        Debugger is active!
        Debugger PIN: 192-688-002
This server will run on http://127.0.0.1:8080/ and is needed for the frontend client to access the card database.

Finally, change directories into the “client” Directory. Once here run the start script by typing “./start.sh”

You can now access the client on http://localhost:8081

Repository Structure

Backend Server:

The backend server directory houses the Restful API that the Frontend Web Client communicates with. The API server is a Restful Flask server written in Python. It makes use of a Card Database class. The server uses “controller.py” as a middleman to communicate with the card database by calling any needed functions to retrieve cards. The Card Database populates itself from a JSON file with data acquired from here.

Frontend Client:

The frontend client directory houses all the code required to run the Web Client that communicates with the Restful API. The functions that control all of the behavior of the main page can be found in the scripts subdirectory in “main.js”. Each of the html files in the directory correspond to a unique page on the client site. “Index.html” hosts the main card searcher tool.

Ways to Contribute

This tool is still in progress! There are a bunch of ways to contribute, and here are just a couple:

  1. Creating updated Test Scripts for the Backend API

  2. Styling frontend to be more compatible with varying screen resolutions and different browsers

  3. Write documentation for the backend API

  4. Add new forms of searching from the base tool

    1. By attributes (Battlecry, Charge, Taunt, etc)

    2. By minion type (Dragon, Murloc, Beast, Mech, etc)

    3. By expansion

    4. By Standard vs Wild

If you have any recommendations for new features feel free to share and implement them!

Please note: When implementing these, make sure to create a pull request to merge it with the master directory.

Visit our github page for ways of contacting us if you have any questions or concerns!
